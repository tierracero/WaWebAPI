# Documentation Sync Skill

Checklist for keeping stable docs aligned with code.

## Update When Applicable

- [ ] `PRODUCT_SCOPE.md` for verified responsibilities/platforms/dependencies.
- [ ] `ARCH_INDEX.md` for new/moved rule ownership.
- [ ] Owning `architecture/*.md` chunk for changed boundaries.
- [ ] `MODULES.md` and `SOURCE_MAP.md` for source-layout changes.
- [ ] `REFERENCE_PROJECTS.md` for dependency ownership changes.
- [ ] `OPEN_DECISIONS.md` for newly open or resolved decisions.
- [ ] `TECH_DEBT.md` for newly verified or repaid debt.
- [ ] `PROJECT_MEMORY.md` for durable state changes.
- [ ] `TASKS.md` / `TASKS_ARCHIVE.md` for tracked work.

## Quality Checks

- [ ] Facts are verified against current source.
- [ ] Plans are not described as implemented.
- [ ] Rules have one owner.
- [ ] Links resolve.
- [ ] No secrets, private payloads, or customer data appear.
- [ ] No filler or stale task diaries remain in stable docs.
